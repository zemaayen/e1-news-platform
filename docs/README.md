## Contributing to Documentation

When adding new documentation:
- Place guides in `/guides` for how-to content
- Place feature docs in `/features` for feature descriptions
- Update this README to link to new documents
- Use clear, descriptive filenames with uppercase and underscores

## Documentation Standards

- Use Markdown format
- Include code examples where applicable
- Add screenshots for UI-related guides
- Keep content up-to-date with code changes
- Use clear headings and sections
